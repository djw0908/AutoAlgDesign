# 算法自动设计平台 (Simplified Algorithm Design Platform)

## 📖 项目简介

本项目是一个面向本科毕业设计需求的简化版算法自动设计平台。它利用大语言模型 (LLM) 的代码生成能力，通过进化策略自动优化和发现解决经典组合优化问题的算法启发式规则。

### 核心功能

1.  **进化策略驱动**：采用进化算法（基于进化策略）不断迭代和改进生成的算法代码。
2.  **多问题支持**：内置 4 个经典算法设计问题：
    *   **在线装箱问题 (Online Bin Packing)**：实时决策物品放入哪个箱子，最小化箱子使用数量。
    *   **旅行商问题 (TSP Construction)**：自动生成构建路径的贪心策略或启发式规则。
    *   **背包问题 (Knapsack Problem)**：设计物品选择的评价函数，优化背包总价值。
    *   **一维装箱问题 (1D Bin Packing)**：传统的离线装箱启发式设计。
3.  **LLM 接入**：支持通过 deepseek API 或自定义 HTTPS API 接入各种大语言模型。
4.  **安全沙箱**：基于进程隔离和临时文件的代码执行沙箱，确保生成的算法代码安全运行并防止超时。
5.  **可视化 GUI**：提供直观的图形用户界面，支持实时查看进化过程、得分趋势及最优算法代码。

## 🚀 快速启动

### 1. 环境准备

确保安装了 Python 3.9+，并安装依赖：

```bash
pip install -r requirements.txt
```

### 2. 运行平台

直接运行根目录下的 `main.py` 启动 GUI 界面：

```bash
python main.py
```

### 3. 配置 LLM

在 GUI 界面中输入您的 LLM API 信息（Host, Key, Model Name），选择算法问题并点击 "Run" 开始自动设计。

## 📂 项目结构

*   `auto_alg/`：核心算法逻辑包
    *   `base/`：基础类定义，包括代码表示、评估接口等。
    *   `method/`：进化算法实现 (`evolution/`)。
    *   `task/`：内置的问题评估模块。
    *   `tools/`：LLM 接口及性能分析工具。
*   `GUI/`：图形界面源代码。
*   `main.py`：项目入口。
*   `test_platform.py`：功能测试脚本。

## 🛠️ 技术特点

*   **进程级沙箱**：使用 `multiprocessing` 实现执行隔离，支持超时控制。
*   **进化算子**：实现了基于 LLM 的交叉 (Crossover) 和变异 (Mutation) 操作。
*   **模块化设计**：易于扩展新的 LLM 接口或新的算法问题。

## 📐 代码规范

### 中文注释规范

本项目要求对所有源代码文件提供一致、清晰、可维护的中文注释说明，用于毕业设计答辩与后续二次开发。

1. 模块级说明
   - 每个源文件顶部必须包含模块级中文说明（三引号 docstring），描述业务背景、职责边界与关键数据流。

2. 类与函数说明
   - 每个类与每个函数/方法必须包含中文 docstring，至少覆盖：功能、参数含义、返回值、异常/边界条件。
   - 对外部可调用的入口函数需补充业务背景与调用约束（例如是否运行于子进程、是否需要线程安全）。

3. 复杂逻辑说明
   - 对关键算法步骤、并发/进程隔离、文件读写协议、序列化格式等复杂逻辑，在其所在函数的 docstring 中明确解释。
   - 若需要更细粒度的逐行说明，优先使用“局部说明块 docstring”（独立三引号字符串表达式）避免破坏代码结构。

4. 常量与配置项说明
   - 对配置项字典、常量列表、正则表达式等，需在其定义邻近位置给出中文说明，明确用途与影响范围。

5. 风格一致性
   - Python 统一使用三引号 docstring 作为注释载体，避免混用多种注释风格导致可读性下降。
   - 注释必须准确、无歧义、无错别字；提交前需完成运行验证与人工走查。
